# 验收分层模型

## 1. 线程内验收
目标：
- 保证当前 prompt 交付物在自己职责边界内正确
- 没有越权改动
- 产出可消费的接口与证据

看什么：
- 改动路径是否受控
- contract 是否满足
- happy path 是否存在
- 必要测试是否通过
- 已知限制是否明确

## 2. Wave 内验收
目标：
- 保证同一 Wave 的多个线程可以拼起来
- 不出现“各自正确，组合错误”

看什么：
- 接口是否冻结
- sample payload 是否一致
- 端到端最小流程是否能通
- invariants 是否被共同满足

## 3. 线程间对接
目标：
- 让相邻线程不靠口头同步，而靠结构化 handoff

看什么：
- 输入输出对象
- 错误模型
- 示例 payload
- 依赖版本/提交
- TODO / blockers

## 4. Wave 间对接
目标：
- 保证下一 Wave 不会建立在模糊前提上

看什么：
- 当前 Wave 的冻结接口
- 未完成项
- 风险项
- 下一 Wave 可依赖的最小真相

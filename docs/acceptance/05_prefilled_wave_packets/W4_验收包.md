# Wave 4 验收包（已预填）

## Wave 4 模块
- emergency_force_close
- shadow_monitor

## 线程内验收重点
### emergency_force_close
- 是否仅在 ActivePosition 可调用
- 是否先 Closed 再执行逃生
- 是否保证迟滞回调随后 Revert

### shadow_monitor
- 是否独立于 Reactive 运行
- 是否使用备用 RPC 对账
- 是否应用 Grace Period
- 是否输出 force-close 建议动作

## Wave 内验收
- Shadow Monitor 发现异常后，CLI 告警文案是否可追溯
- force-close 后正常迟滞回调是否被挡住
- 是否没有把 monitor 变成第二执行引擎

## 通过标准
- 至少一条“该死却没死”的检测用例
- 至少一条 force-close 成功用例
- 至少一条 force-close 后迟滞回调 revert 用例

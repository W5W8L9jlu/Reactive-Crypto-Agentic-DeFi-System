# Wave 1 验收包（已预填）

## Wave 1 模块
- provider_architecture
- export_outputs
- strategy_boundary_service
- validation_engine
- investment_state_machine_contract

## 线程内验收重点
### provider_architecture
- 是否只落地 Phase 1 最小 provider：RPC / The Graph / Etherscan fallback
- 是否有统一 shared http client
- 是否避免过早接入 Bitquery / Moralis 主依赖

### export_outputs
- Machine Truth / Audit / Investment Memo 是否三轨分离
- Audit 是否只摘抄不改写
- Memo 是否不反向污染执行对象

### strategy_boundary_service
- 是否只负责规则边界与三分流
- 是否没有偷偷做链上状态确认

### validation_engine
- 是否基于 Pydantic v2
- 是否没有散落 schema if/else

### investment_state_machine_contract
- 是否已定义 PendingEntry / ActivePosition / Closed
- 是否有 registerInvestmentIntent / executeReactiveTrigger 的最小骨架

## Wave 内验收
- provider 接口是否能被后续 context builder 消费
- strategy boundary 和 validation 的对象命名是否一致
- 输出层命名是否与后续 CLI / adapter 可对齐
- 合约状态机原语是否稳定，不会在 Wave 2 大改

## 通过标准
- 5 个模块都提交 thread acceptance + handoff
- 至少有 provider smoke test
- 至少有 export format 示例
- 至少有 contract compile/test 证据
- 共享 schema 名称冻结

# Wave 3 验收包（已预填）

## Wave 3 模块
- cryptoagents_adapter
- execution_layer
- reactive_runtime
- cli_surface

## 线程内验收重点
### cryptoagents_adapter
- 是否保留多 agent orchestration 骨架
- 是否输出结构化对象，而不是自由文本执行建议
- 是否不生成最终 calldata

### execution_layer
- 是否只消费已编译计划和运行时触发
- 是否不在 validation 后立刻 swap
- 是否产出 ExecutionRecord

### reactive_runtime
- 是否只做事件驱动、条件触发、callback
- 是否没有塞入自由决策或复杂业务逻辑
- 是否适配入场 + 止损止盈

### cli_surface
- 是否只做命令路由、展示、人工入口
- 是否未承载核心业务计算

## Wave 内验收
- CLI dry-run 是否能串起 context -> adapter -> validation/approval
- 执行链是否符合 `Validation -> PRC -> Compiler -> register -> ISM -> Runtime Check -> Execution`
- reactive adapter 与 execution layer 是否职责分离
- Approval / Export 命令面是否与前两 Wave 的结构化对象兼容

## 通过标准
- 至少一条 dry-run 主流程可演示
- 至少一条注册后触发的最小执行链可验证
- CLI 命令面可调用到正确服务层

# Wave 2 验收包（已预填）

## Wave 2 模块
- decision_context_builder
- pre_registration_check
- execution_compiler
- approval_flow

## 线程内验收重点
### decision_context_builder
- 是否输出统一 DecisionContext
- 是否输入视角上移到趋势 / 资金流 / 风险环境 / 组合状态

### pre_registration_check
- 是否只信 RPC 作为注册前真相
- 是否有 gas / expected profit / ttl / allowance / reserve 检查
- 是否明确 abort 条件

### execution_compiler
- 是否只在注册时编译
- 是否固化 entry minOut / entry TTL / maxGasPrice
- 是否对 stop/take 写入 slippage bps，而不是触发时重新编译

### approval_flow
- 是否由结构化对象映射到 ApprovalBattleCard
- 是否默认不暴露 raw JSON
- TTL 过期是否不可审批

## Wave 内验收
- DecisionContext 是否能喂给 adapter（即便 adapter 尚未实现完整）
- precheck 输出对象是否能被 compiler 稳定消费
- compiler 产物字段是否能驱动 approval 展示
- bps / ttl / timestamp 单位是否统一

## 通过标准
- 一个注册前 happy path 可以走通到 `ExecutionPlan`
- 一个审批展示 happy path 可以从 ValidationResult/TradeIntent/ExecutionPlan 渲染
- 预检查失败路径可被明确中止

# P2_W1 Wave Gate

## Wave Goal

Prove the offline Core Execution Loop using fixtures only. W1 should run without RPC or contract execution.

## Scope

```text
TradeIntent fixture
-> Validation
-> PreRegistrationCheck fixture result
-> ExecutionPlan
-> JSON / Audit Markdown / Investment Memo export
```

## Allowed Paths

- Phase2 schema / validation paths listed in prompts
- Phase2 compiler paths listed in prompts
- Phase2 export paths listed in prompts
- `fixtures/**`
- `docs/acceptance/threads/phase2_wave1/**`

## Forbidden Paths

- Solidity state machine implementation unless explicitly scoped to W1 fixture compatibility
- live RPC calls
- Reactive callback implementation
- Phase3 approval queue
- Shadow Monitor daemon
- Aave / Uniswap V3 / cross-chain implementation

## Required Contracts

- `docs/contracts/phase2_interface_freeze.contract.md`
- `docs/contracts/phase2_core_execution_loop.contract.md`
- `docs/contracts/phase2_disabled_features.contract.md`
- `docs/contracts/validation_engine.contract.md`
- `docs/contracts/execution_compiler.contract.md`
- `docs/contracts/export_outputs.contract.md`

## Required Smoke Tests

```powershell
python scripts/workflow.py check validation_engine --execute --strict
python scripts/workflow.py check execution_compiler --execute --strict
python scripts/workflow.py check export_outputs --execute --strict
```

Implementation-specific smoke should prove fixture `TradeIntent` produces a deterministic `ExecutionPlan` and matching export files.

## Exit Criteria

- [ ] valid long-only V2 fixture passes validation.
- [ ] cross-chain fixture fails fast.
- [ ] Uniswap V3 fixture fails fast.
- [ ] approval-required fixture aborts registration.
- [ ] ExecutionPlan is deterministic.
- [ ] JSON / Audit Markdown / Memo roles remain separated.

## Delivery Evidence

- `docs/acceptance/threads/phase2_wave1/offline_core.delivery_note.md`
- `docs/acceptance/threads/phase2_wave1/offline_core.test_evidence.md`
- `docs/acceptance/threads/phase2_wave1/offline_core.thread_acceptance.md`

## Rollback / No-Go Conditions

- W1 requires live RPC.
- W1 mutates W0 frozen schema without an interface change request.
- Audit Markdown becomes a truth source.
- disabled features silently pass.

## Handoff to Next Wave

W2 may use W1's deterministic fixture `ExecutionPlan` as the local-chain registration input.

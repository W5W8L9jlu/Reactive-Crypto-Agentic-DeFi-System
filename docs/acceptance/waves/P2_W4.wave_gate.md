# P2_W4 Wave Gate

## Wave Goal

Close Phase2 with ReactiveExecutorAdapter v1, hardening, disabled feature tests, and final export evidence.

## Scope

- ReactiveExecutorAdapter v1 callback adapter
- LocalExecutor baseline preserved
- idempotent event sync
- domain-specific failure handling
- disabled feature fast-fail tests
- JSON / Audit Markdown / Investment Memo closure
- Phase2 Go/No-Go evidence

## Allowed Paths

- reactive adapter paths listed in prompts
- event syncer hardening paths listed in prompts
- export paths listed in prompts
- disabled feature tests
- `docs/acceptance/threads/phase2_wave4/**`

## Forbidden Paths

- full approval queue implementation
- Shadow Monitor daemon implementation
- Aave implementation
- Uniswap V3 implementation
- cross-chain implementation
- webhook implementation beyond disabled interface behavior

## Exit Criteria

- [ ] ReactiveExecutorAdapter v1 can trigger the same state transitions as LocalExecutor in the supported path.
- [ ] LocalExecutor remains available for fork/testnet debugging.
- [ ] EventSyncer is idempotent.
- [ ] disabled feature tests cover approval queue, shadow daemon, Aave, V3, cross-chain, and webhook.
- [ ] JSON is Machine Truth.
- [ ] Audit Markdown only excerpts Machine Truth.
- [ ] Investment Memo does not mutate execution parameters.
- [ ] Phase2 Go/No-Go can be evaluated from recorded evidence.

## Rollback / No-Go Conditions

- Reactive callback performs free strategy decisions.
- disabled feature path reaches registration.
- export outputs disagree with Machine Truth.
- Phase2 Go/No-Go cannot trace evidence to W0-W4 gates.

## Handoff to Next Phase

Phase 3 may start only after Phase2 Go/No-Go is `GO`. Phase 3 owns Approval Flow, Shadow Monitor daemon, operational safety, and richer alerting.

# P2_W0 Wave Gate

## Wave Goal

Freeze all interfaces required for Phase2 parallel development. W1-W4 agents must consume these contracts instead of inventing their own schema, ABI, events, fixtures, errors, or feature flags.

## Scope

- Pydantic truth models
- JSON Schema export
- Solidity ABI and events
- `ExecutionRecord` / intent mapping persistence shape
- CLI command skeleton
- fixtures
- error taxonomy
- disabled feature defaults

## Allowed Paths

- `docs/knowledge/00_meta/03_phase2_wave_parallelization_map.md`
- `docs/knowledge/08_delivery/05_phase2_wave_plan.md`
- `docs/knowledge/09_testing/04_phase2_wave_smoke_tests.md`
- `docs/contracts/phase2_interface_freeze.contract.md`
- `docs/contracts/phase2_core_execution_loop.contract.md`
- `docs/contracts/phase2_disabled_features.contract.md`
- `docs/acceptance/waves/P2_W0.wave_gate.md`
- future W0 implementation paths explicitly listed in module prompts

## Forbidden Paths

- unrelated Phase1 acceptance evidence
- Phase3/Phase4 implementation paths
- production deployment configuration
- secret or environment files

## Required Contracts

- `docs/contracts/phase2_interface_freeze.contract.md`
- `docs/contracts/phase2_core_execution_loop.contract.md`
- `docs/contracts/phase2_disabled_features.contract.md`

## Frozen Interfaces

The following must be frozen before W1 starts:

- `StrategyTemplate`
- `TradeIntent`
- `ValidationResult`
- `PreRegistrationCheckResult`
- `ExecutionPlan`
- `InvestmentIntentPayload`
- `ExecutionRecord`
- `FeatureFlags`
- Solidity `InvestmentIntent`
- `IntentRegistered`, `EntryExecuted`, `ExitExecuted`, `IntentExpired`, `IntentCancelled`, `EmergencyForceClosed`, `TriggerSkipped`
- Phase2 error taxonomy

## Required Smoke Tests

```powershell
python scripts/workflow.py audit-manifest --strict
python scripts/workflow.py check validation_engine --execute --strict
python scripts/workflow.py check execution_compiler --execute --strict
```

If implementation paths do not exist yet, record the skipped command and reason in W0 test evidence.

## Exit Criteria

- [ ] W0 contracts exist and are reviewed.
- [ ] schema, ABI, events, fixtures, errors, and feature flags are listed.
- [ ] downstream W1-W4 prompts can reference frozen contracts.
- [ ] disabled features default to disabled.
- [ ] any skipped checks are documented with a follow-up owner.

## Rollback / No-Go Conditions

- W0 allows multiple schema or ABI variants.
- W0 omits disabled feature behavior.
- later Waves start implementation before frozen interfaces exist.

## Handoff to Next Wave

W1 may start only after W0 establishes the frozen interfaces and fixture names. W1 consumes W0 fixtures and must not redefine them.

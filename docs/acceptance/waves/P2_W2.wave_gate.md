# P2_W2 Wave Gate

## Wave Goal

Prove the local chain / mock DEX state machine from registration through close.

## Scope

```text
registerInvestmentIntent
-> tokenIn custody
-> PendingEntry
-> executeReactiveTrigger entry
-> ActivePosition
-> executeReactiveTrigger stop-loss or take-profit
-> Closed
```

## Allowed Paths

- Phase2 contract implementation paths listed in prompts
- local executor paths listed in prompts
- local/mock DEX fixtures
- Foundry/local chain tests
- `docs/acceptance/threads/phase2_wave2/**`

## Forbidden Paths

- live mainnet execution
- cross-chain execution
- Uniswap V3 implementation
- Aave implementation
- Shadow Monitor daemon
- complete approval queue

## Required Contracts

- `docs/contracts/phase2_interface_freeze.contract.md`
- `docs/contracts/phase2_core_execution_loop.contract.md`
- `docs/contracts/investment_state_machine_contract.contract.md`
- `docs/contracts/phase2_price_oracle_adapter.contract.md`

## Required Smoke Tests

```powershell
python scripts/workflow.py check execution_layer --execute --strict
python scripts/workflow.py check reactive_runtime --execute --strict
```

Implementation-specific smoke should include Foundry or local-chain tests for register, entry, exit, closed-state repeat trigger, and emergency force-close availability.

## Exit Criteria

- [ ] registration transfers `tokenIn` custody.
- [ ] initial state is `PendingEntry`.
- [ ] entry trigger records `actualPositionSize`.
- [ ] entry moves state to `ActivePosition`.
- [ ] stop-loss or take-profit exit records amount out.
- [ ] exit moves state to `Closed`.
- [ ] repeat trigger on `Closed` reverts or fails with a domain error.
- [ ] required events are emitted.

## Rollback / No-Go Conditions

- custody model changes from register-time `tokenIn` custody.
- contract relies on off-chain free decisions during callback.
- state machine allows repeat execution after `Closed`.

## Handoff to Next Wave

W3 may replace mock chain inputs with fork/testnet RPC only after W2 proves the local state machine.
